Трёхступенчастая (клиент --> сервер --> база данных)

Клиент -->

Клиент -->	Сервер --> База данных

Обычно ставят несколько сервер, на случай, если с одним из них что-то случится. Так же может присутствовать балансировщик, который распределяет, на какой сервер лучше всего делать запрос.

Клиент -->
Клиент -->	Сервер -->
Клиент -->			База данных
Клиент -->	Сервер -->
Клиент -->

// -----------------------

Клиент -->
Клиент -->					-->	Сервер -->
Клиент -->	балансировщик(распределитель) 				База данных
Клиент -->					-->	Сервер -->
Клиент -->

Функция балансировщика выбрать, более разгруженый сервер и сделать туда запрос, равномерна распределяя загруженость.

Иногда загружают только один сервер, а второй просто как подстраховка, в случае ошибок на первом.
